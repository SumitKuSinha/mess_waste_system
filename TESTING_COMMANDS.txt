TEST 11: TEST FAILOVER (HIGH AVAILABILITY)
================================================================================

STEP 1: Find one of the Auth service windows (Auth-1 on port 5001)
STEP 2: Press Ctrl+C to stop it (kill that instance)
STEP 3: The window shows "[Stopped]" or similar

STEP 4: Run this command (while Auth-1 is down):

for ($i = 1; $i -le 6; $i++) { Write-Host "Request $i (Auth-1 DOWN):" -ForegroundColor Red; curl http://localhost:8080/api/auth/health; Start-Sleep -Milliseconds 500 }

EXPECTED: All 6 requests STILL WORK!
Requests go to: 5011 and 5021 only (skipping the dead 5001)

This proves FAILOVER IS WORKING!

STEP 5: Restart the service by typing in that same window:

$env:PORT=5001; npm start

Wait 10-15 seconds for it to restart

STEP 6: Run requests again:

for ($i = 1; $i -le 9; $i++) { Write-Host "Request $i (Auth-1 RESTARTED):" -ForegroundColor Cyan; curl http://localhost:8080/api/auth/health; Start-Sleep -Milliseconds 500 }

EXPECTED: Now requests cycle through all 3 ports again (5001, 5011, 5021)

This proves SERVICE REJOIN IS WORKING!

TROUBLESHOOTING
================================================================================

Port Already in Use:

Get-NetTCPConnection -LocalPort 5001 | Select OwningProcess
Get-Process -Id [PID-FROM-ABOVE]
Stop-Process -Id [PID] -Force

MongoDB Not Running:
Get-Service MongoDB
(If it says "Stopped", start it)

Redis Not Running:
tasklist | findstr redis
(If not there, start Redis server)
